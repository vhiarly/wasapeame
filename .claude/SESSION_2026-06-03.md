## Script de referencia — onboarding saliente

```python
import os, time
from dotenv import load_dotenv
load_dotenv("/Users/vhiarly/Desktop/wasapeame/.env")
from twilio.rest import Client

client = Client(os.getenv("TWILIO_ACCOUNT_SID"), os.getenv("TWILIO_AUTH_TOKEN"))
FROM = "whatsapp:+18492659906"
TO   = "whatsapp:+DESTINATARIO"

# Google Drive URL de descarga directa (no el visor)
PDF_URL = "https://drive.google.com/uc?export=download&id=FILE_ID"

parte1 = "..."   # max 1600 chars — datos del negocio
parte2 = "..."   # max 1600 chars — comandos admin
# Si hace falta parte3 (link de compartir), añadir con otro sleep(2)

m1 = client.messages.create(body=parte1, from_=FROM, to=TO)
print(f"Parte 1 — {m1.sid} ({m1.status})")
time.sleep(2)

m2 = client.messages.create(body=parte2, from_=FROM, to=TO)
print(f"Parte 2 — {m2.sid} ({m2.status})")
time.sleep(2)

m3 = client.messages.create(body="", media_url=[PDF_URL], from_=FROM, to=TO)
print(f"PDF    — {m3.sid} ({m3.status})")

# Polling hasta confirmación
FINALES = {"sent", "delivered", "failed", "undelivered"}
for i in range(40):
    time.sleep(3)
    estado = client.messages(m3.sid).fetch().status
    print(f"  [{i+1}] {estado}")
    if estado in FINALES:
        break
```

## Reglas de envío aprendidas en producción

1. `from_` siempre `whatsapp:+18492659906`
2. Límite 1,600 chars por mensaje — fragmentar con `time.sleep(2)` entre partes
3. Media: usar Google Drive con `?export=download&id=FILE_ID`, nunca el link del visor
4. Ventana 24h de Meta — onboarding solo dentro de la primera sesión del cliente
5. Kudu VFS (SCM) tiene basic auth deshabilitada en este App Service — no funciona para subir archivos
